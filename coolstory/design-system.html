How to Build a Page

Page anatomy

1 · <nav class="nav"> — every page
2 · <section class="hero"> — home only
2 · <div class="section-header"> — other pages
3 · content <section>s — any number
4 · <section class="cta-section">
5 · <footer class="footer"> — every page

Never do these

✗ Write new CSS or create new class names

✗ Use inline styles (except background/padding on <section> shells)

✗ Use hex colors anywhere in HTML

✗ Reference internal tokens (--c-green, --c-orange …) in HTML

✗ Use <h1> outside the hero section

✗ Set font-size, font-family, or font-weight inline

✗ Nest .container inside another .container

✗ Use .grid-3--2-mobile on text-heavy cards

Section background rule — always alternate, never repeat

--section-bg-1

Product & feature sections. Blue-tinted gradient.

--section-bg-2

Utility & informational sections. Grey gradient.

--c-white

Sections with strong card visuals: gradients, integrations, pricing. Use sparingly.

Content section shell — copy this pattern

<section style="background: var(--section-bg-1); padding: var(--section-py) 0;">
  <div class="container">
    <div class="section-header">
      <h2>Section Title</h2>
      <p>Optional subheading.</p>
    </div>
    <!-- grid or component goes here -->
  </div>
</section>

.grid-4 — icon tiles, short-text cards (4 columns)
